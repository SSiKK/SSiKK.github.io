Description

    Here's a description about our CS422 Project 2: Knockin' on Heaven's Door.
    The application is called Doorway: Your Door Your Way created by Kyle, Shiwangi, Krishna and Saumya.

    Doorway is a Interface Design of a door, both inside and outside which is laid out on a Canvas. You can think of the interface as a high-tech door. This application is a user interface demonstrating various functionalies that a door can have like locking/unlocking, camera to talk to people at your door, notifications, security for your home and other decorating features designed on a JavaScript Canvas. This page will provide you with the features of the Doorway application and instructions on how to use the application. So, let's get started!

    How to use it?

    State of the door (locked or unlocked)

    Initial state of the locking/unlocking mechanism: On the inside door, you have a Red bolt feedback suggesting that the door is locked. When a user from outside unlocks the door, a feedback in the form of a Green ring around the doorknob is shown and the bolt turns Green too. When the user from the outside locks the door, a feedback in the form of a Red ring is shown on the outside and the bolt turns Red too. This feddback ring stays only for few seconds and then disappears.

    Ability to lock or unlock from inside

    User can click on the bolt which can lock or unlock the door from inside. A feedback in the form of color change from Green -> Red and from Red -> Green is sent to the user for locking and unlocking respectively.

    Menu System

    Doorbell chime

    Part of the menu system is Chimes. When you click on the doorbell icon, chimes window appears. This has a checkbox control. You can select the desired chime and the next time somebody on your door presses a bell, you will have the desired chime for the doorbell.

    Ability to write message on inside door for General/Specific people

    Talk to people on the outside with an avatar

    Ability to lock or unlock from outside with a key

    User profiles: Talking about the users, we have saved/can save user profiles with id, name, handprint, passcode and image information. So, if the user has a handprint and a passcode saved in the system, he/she can unlock the door.
    Locking/unlocking from the outside is done by using two mechanisms in our system:
    1. By Handprint: In our interface, we have placed a handprint area just besides the door knob at a reachable height. User from the outside if registered in the system, can unlock the door by placing their palm in the provided area.
    2. By passcode: Just besides the handprint area is a passcode area. User can press the button and enter the passcode from their phones. If the phone is registered with the interface, the user can unlock the door.

    'Push' the doorbell

    When the doorbell on the outside is pressed, you can hear a 'ting' sound. There is a doorbell icon in our interface design right above the doorknob.
