Round 1

Just a little commentary about the thinking process here...this was what I did!



Locked state is represented by Red color.



Unlocked state is represented by Green color.



Registered Users can use Phone protected password to interface with the door. The key icon turns green as you do send a message to the door.



As you enter the password from your phone, the key icon turns green. You need to press this button.



The door is now unlocked.



Your voice is fed into the system when you register as a User with this high tech door. Your voice recognition is done by the system which unlocks the door.



I have set an alternative mechanism of opening the door just incase you don't have your phone with you and you want to unlock the door.



Based upon the proximity, if a general person comes closer to the door, a message is displayed on the outside door. The proximity is set to 2m Radius.



If the person walking by the door is a registered user, the phone is used to detect his/her presence. A personalized message is displayed. The proximity is set to 10m Radius.



This is how the default inner interface looks like. The weather, reminders and sticky notes are always displayed for the user leaving the door.



The organization of the page is such that the lock is a little lower for even kids to reach.



Menu icon and user icons are present always. Users can be added or modified, we will see this later in the sketches.

p>

I wanted to keep an explicit icons for video and voice calls, just to make it simple for users to interface for such basics operations. They do not have to go to Menu icon for this.



Weather API with Date, time and location will be displayed on top always.



Reminders are the important notifications that any person leaving the house can see and take a glance at. These reminders will be synced with the phone.



The other last thing is sticky notes. We will discuss this later in detail, although the messages are displayed to the person outside and inside via sending sticky notes from either side. A little pen icon can be used to choose pen and send icon for sending messages outside.



Green color represents Unlocked state.



I have put a small callout tutorials for making it easy for a new user to interface with the door.



Register users: Touchning the icon will show you which users are so far registered. They include family members, guests and adding a new user.



You can view the information of the registered users. A green check mark means that feature is incorporated in the user profile.



Add User: Touching the plus sign would show up the form where you can register information of the new users. Main information includes phone details and voice recognition for 'Unlocking' the door.



For an individual who is outside the door, he can use the sticky note to leave a message inside the door. You can see the send icon the outside door is sending message inwards. I have actually overlayed the sticky notes, the most recent message is displayed on top.



You can talk to the person inside from outside. At this point the microphone icon is turned green on both sides.



The person inside can be viewed from outside on a screen . This featured incorporates Turn Off/On. You may not want to show what you are doing inside so you can just disble. The icon turns Red/Green.



When you want the camera to turn on, just press the icon, it turns red and you can see some default image on the outside.



This is the menu that I have created for all the functionalities for life size display, safety, chimes, themes for the doors Outside and Inside, mirror, talking to avatar. All these features are nicely encapsulated in a 3 X 3 matrix.
